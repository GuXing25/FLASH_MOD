# FLASH_MOD 配置模板

本文件保留整合版的完整字段表；在 `FLASH_MOD_DIV/NAND` 中，实际有效的
`FLASH_CLASS` 只允许 `SPI_NAND` / `SPINAND` / `RAW_NAND` / `NAND`，NOR
相关字段仅作为模板参考。

配置格式：

```text
KEY VALUE
KEY = VALUE
```

`#` 后面是注释。数字支持十进制和 `0x` 十六进制。ID 字节可以写成：

```text
JEDEC_ID EF 40 16
```

## 基本字段

```text
FLASH_CLASS SPI_NAND | SPINAND | RAW_NAND | NAND
CHIP_NAME <string>
STORAGE_FILE <path>
JEDEC_ID <byte...>
RESET_STORAGE_ON_START 0|1
AUTO_COMPLETE 0|1
USE_MAX_TIME 0|1
```

## NOR 几何

```text
MEMORY_SIZE <bytes>
PAGE_SIZE 256
SECTOR_SIZE 4096
BLOCK32_SIZE 32768
BLOCK64_SIZE 65536
ADDRESS_BYTES 3|4
WRAP_ADDRESS 0|1
PAGE_PROGRAM_WRAP 0|1
```

## SPI-NAND 几何

```text
MAIN_SIZE 2048
SPARE_SIZE 64
PAGE_SIZE 2112
PAGES_PER_BLOCK 64
BLOCKS 1024
PLANES 1
BLOCKS_PER_PLANE 1024
```

如果 `PAGE_SIZE` 没有显式覆盖，会按 `MAIN_SIZE + SPARE_SIZE` 派生。

## ECC / Spare

```text
ECC_SUPPORTED 0|1
ECC_ENABLE 0|1
ECC_RESERVED_START 0x840
ECC_RESERVED_END 0x880
GENERATE_INTERNAL_ECC_PARITY 0|1
BAD_BLOCK_MARKER_COLUMN 2048
```

`ECC_RESERVED_START/END` 是半开区间 `[start, end)`。ECC 开启时，`program_load()` 如果写入该区间，`program_execute()` 会置 `P_FAIL` 并失败。

## 状态与 Feature 默认值

```text
STATUS1_DEFAULT 0x00
STATUS2_DEFAULT 0x02
STATUS3_DEFAULT 0x00
FEATURE_A0_DEFAULT 0x7C
FEATURE_B0_DEFAULT 0x10
FEATURE_D0_DEFAULT 0x00
FEATURE_F0_DEFAULT 0x00
```

SPI-NAND 里：

- `A0h` 通常表示 block lock。
- `B0h` 通常表示 ECC/QE/OTP/配置位。
- `C0h` 由模型动态映射为状态寄存器。

## 时序

```text
F_C_MHZ 133
F_R_MHZ 50
F_Q_MHZ 80
T_PUW_US 5000
T_READ_ECC_OFF_US 25
T_READ_ECC_ON_US 50
T_PP_US 400
T_PP_MAX_US 3000
T_SE_US 45000
T_BE32_US 120000
T_BE64_US 150000
T_ERASE_US 2000
T_ERASE_MAX_US 10000
T_CE_US 10000000
T_RESET_US 30
```

兼容别名：`F_CLOCK_MHZ`、`F_SPI_MHZ`、`T_PROG_US`、`T_BERS_TYP_US` 等会被解析到统一字段。

## 设计建议

上层从 datasheet 提参时，优先只产出这些字段。若某个厂商差异无法表达，再扩 schema，而不是复制一个新的 `flash_core.cpp`。
