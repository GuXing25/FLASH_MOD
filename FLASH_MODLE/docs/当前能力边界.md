# 当前能力边界

## 已实现的最小公共路径

- 配置加载：读取分层 YAML-like 配置。
- 配置验证：区分 NOR 与 SPI-NAND 必填字段。
- 模型构建：根据 `policy.name` 和 `capabilities` 组合模型。
- 地址映射：`AddressMapper` 统一负责 NOR sector 对齐、保护范围重叠、NAND page/block offset。
- 寄存器状态：`RegisterEngine` 统一负责 WEL、SR1/SR2/SR3、Feature A0/B0/C0/D0、P_FAIL/E_FAIL。
- 时序状态：`TimingEngine` 统一负责 now_us、busy 窗口、wait-ready、普通时间推进。
- NOR：
  - read-id
  - write-enable
  - sector erase
  - page program
  - direct read
  - `old & new` 编程语义
  - wait-ready 时间推进
  - 通用保护窗口 `nor_protect_start/nor_protect_length`
- SPI-NAND：
  - read-id
  - get/set feature
  - block erase
  - program load
  - program execute
  - page read
  - read from cache
  - `old & new` 编程语义
  - ECC reserved 区间检查
  - partial program 次数检查
  - strict sequential program 检查

## 已进入 smoke test 的能力行为

- `block_protect`：demo SPI-NAND 默认 A0h 全保护，未解锁前 block erase 必须失败。
- `block_protect`：demo NOR 保护窗口内 program/erase 必须失败，窗口外正常执行。
- `ecc_status`：demo SPI-NAND 写入 ECC reserved spare 区后，program execute 必须失败。
- `strict_sequential_program`：demo SPI-NAND 跳过 page 1 直接 program page 2 必须失败。
- `validator`：NOR 配置若启用 NAND-only 命令，必须在构建模型前失败。

- 逐 SCLK/SPI waveform 仿真。
- 电压、电流、温度、封装、电气波形。
- 完整 ECC 算法。
- 完整 FTL、磨损均衡、垃圾回收。
- 所有厂商私有命令的完整覆盖。
