# 当前能力边界

## 已实现的最小公共路径

- 配置加载：读取分层 YAML-like 配置。
- 配置验证：区分 NOR 与 SPI-NAND 必填字段。
- 字段证据：`evidence` 段支持 `field.path: datasheet|page|confidence`。
- evidence gap：validator 会报告关键字段缺少来源证据，但不阻止模型构建。
- 模型构建：根据 `policy.name` 和 `capabilities` 组合模型。
- 地址映射：`AddressMapper` 统一负责 NOR sector 对齐、保护范围重叠、NAND page/block offset。
- 寄存器状态：`RegisterEngine` 统一负责 WEL、SR1/SR2/SR3、Feature A0/B0/C0/D0、P_FAIL/E_FAIL。
- 时序状态：`TimingEngine` 统一负责 now_us、busy 窗口、wait-ready、普通时间推进。
- NOR：
  - read-id
  - write-enable
  - sector erase
  - block erase
  - chip erase
  - page program
  - direct read
  - `old & new` 编程语义
  - page program 页内回绕
  - wait-ready 时间推进
  - 通用保护窗口 `nor_protect_start/nor_protect_length`
  - Deep Power-Down / Release
  - 3 个 256B Security Register 的 read/erase/program
- SPI-NAND：
  - read-id
  - get/set feature
  - block erase
  - program load
  - program execute
  - page read
  - read from cache
  - `old & new` 编程语义
  - ECC reserved 区间检查
  - initial bad block 拒绝擦除和编程
  - copy-back：从源页读入内部 cache，并按 program 约束提交到目标页
  - partial program 次数检查
  - strict sequential program 检查

## 已进入 smoke test 的能力行为

- `block_protect`：demo SPI-NAND 默认 A0h 全保护，未解锁前 block erase 必须失败。
- `block_protect`：demo NOR 保护窗口内 program/erase 必须失败，窗口外正常执行。
- `deep_power_down`：demo NOR 进入 DPD 后普通 read 必须失败，release 后恢复。
- `security_register`：demo NOR security register 擦除、编程、读回和二次编程 `old & new` 必须通过。
- `ecc_status`：demo SPI-NAND 写入 ECC reserved spare 区后，program execute 必须失败。
- `bad_block_management`：demo SPI-NAND 对 initial bad block 执行 erase/program 必须失败。
- `copy_back`：demo SPI-NAND 可将 page 0 copy 到 page 1，并保持 `old & new` 后的数据。
- `strict_sequential_program`：demo SPI-NAND 跳过 page 1 直接 program page 2 必须失败。
- `validator`：NOR 配置若启用 NAND-only 命令，必须在构建模型前失败。
- `datasheet_tq/PDF profiles`：BY25Q64AS、GD25LE128E、GD5F1GM7UEYIGR、M25P40、MX25L25645G、W25Q32JV、W25N01GV、MT29F2G01 八个配置必须通过 validate-only。

## 已模块化的 capability 行为

- `block_protect` 已经从 `FlashModel` 直接判断迁移到 `BlockProtectModule`。
- `ecc_status` 已经从 `FlashModel` 直接判断迁移到 `EccStatusModule`。
- `bad_block_management` 已经从能力占位推进到 `BadBlockModule`，支持配置初始坏块并拒绝 erase/program。
- `copy_back` 已经从能力占位推进到 `CopyBackModule`，并由 `FlashModel::copy_back` 复用公共 program 提交约束。
- `FlashModel` 现在通过 capability hook 在命令执行前询问模块是否允许：
  - `before_nor_program`
  - `before_nor_erase`
  - `before_nand_block_erase`
  - `before_nand_program_execute`
- SPI-NAND `program_load` 会通过 `marks_nand_program_load_violation` 询问 capability 是否触碰了 reserved 区。

## 已有骨架但尚未完整实现

- OTP。
- Suspend/Resume。
- Die Select。
- Plane Select。
- Read Retry。
- 厂商 family policy 的复杂私有行为。

## 明确不在第一版中追求

- 逐 SCLK/SPI waveform 仿真。
- 电压、电流、温度、封装、电气波形。
- 完整 ECC 算法。
- 完整 FTL、磨损均衡、垃圾回收。
- 所有厂商私有命令的完整覆盖。
